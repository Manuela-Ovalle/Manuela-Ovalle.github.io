Javascript Fundamentals

How do HTML, CSS and Javascript work go together?

Javascript is a tool that allows a web developer the ability to take a static website and make it dynamic and interactive. As I was thinking of an analogy to illustrate this I thought of directions and maps as a way to illustrate the relationship between HTML, CSS and JS (javascript) in the process of building an app.

HTML is like having a step by step list of instructions for a guide:

- Go to the front door of the house
- drive to the 200m to the end of Apple street and turn right onto Kale lane
- drive 50m on Kale lane and turn right onto Avo avenue
- drive 120m on Avo ave and turn left onto Cherry Street
- drive to the end of Pizza road (130m) and turn left on Pineapple place
- drive 50m down pineapple place and turn right onto Cheese ave
- drive for 220m down Cheese ave and you’ve reached your destination!

CSS takes the instructions from HTML
and makes them visual, like a road map book:

Now we can see what the journey will look like but our map is static.

JAVASCRIPT takes our map a step further

Our guide is not static anymore, now it’s like using google maps. Instead of having one journey that we must follow (hard code) google maps allows us to go a different route to reach our journey, if we need to. Javascript gives us the ability to make changes to our HTML elements both manually but also with code that reacts to the actions of a user. If I take a wrong turn, the map allows for an alternate route.
